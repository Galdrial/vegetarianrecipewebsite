# Contributing Guidelines for Green Eats

Thank you for your interest in contributing to this project!

## How to contribute

1. Fork the repository
2. Create a branch for your feature or fix (`git checkout -b feature-name`)
3. Commit your changes (`git commit -m 'Add new feature'`)
4. Push the branch (`git push origin feature-name`)
5. Open a Pull Request

## Style rules
- Follow the existing folder structure
- Use clear variable and component names
- Write comments where necessary
- Ensure the site is responsive
- Run `npm run lint` before submitting your PR

## Reporting bugs
Open an Issue describing the problem and how to reproduce it.

## Questions
For any questions, open an Issue or contact the maintainers.
